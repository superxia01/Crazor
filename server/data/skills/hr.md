# 人事管理 Skill

> 本 Skill 通过 MCP 文档工具管理全部人事数据。所有员工档案、考勤记录、绩效评估、招聘信息均以文档形式存储在知识库中。

---

## 1. 可用 MCP 工具

| 工具 | 用途 | 关键参数 |
|------|------|----------|
| **create_doc** | 创建新文档 | `scope="knowledge"`, `title`, `content` |
| **update_doc** | 更新已有文档 | `id`, `title`, `content` |
| **read_doc** | 读取单个文档 | `id` |
| **list_docs** | 列出文档列表 | `scope="knowledge"`, `folder_id` |
| **search_docs** | 搜索文档 | `scope="knowledge"`, `q` |
| **read_vault_file** | 读取参考模板文件 | 文件路径 |

---

## 2. 知识库文档结构

所有人事文档存储在 `业务流程/人事管理/` 文件夹下：

| 文档类别 | 存储路径 | 文档标题格式 |
|----------|----------|-------------|
| 员工档案 | `业务流程/人事管理/` | `员工档案-姓名` |
| 考勤记录 | `业务流程/人事管理/` | `YYYY年MM月考勤记录` |
| 绩效评估 | `业务流程/人事管理/` | `YYYY年QN绩效评估` |
| 招聘管理 | `业务流程/人事管理/` | `招聘-岗位名称` / `候选人追踪` / `面试记录` |
| 薪资管理 | `业务流程/人事管理/` | `薪资标准` / `YYYY年MM月薪资发放记录` |
| 人事索引 | `业务流程/人事管理/` | `人事索引` |

> **搜索技巧**：使用 `search_docs(scope="knowledge", q="关键词")` 快速定位文档。例如搜索某员工的所有相关文档。

## 3. 工作流程

### 3.1 新员工入职

```
步骤 1：收集员工基本信息（姓名、部门、职位、联系方式等）
步骤 2：create_doc(
          scope="knowledge",
          title="员工档案-{{姓名}}",
          content=<员工档案模板内容，填入实际数据>
        )
步骤 3：read_doc(id=人事索引文档ID)
步骤 4：update_doc(id=人事索引文档ID, content=<追加新员工记录>)
步骤 5：记录合同信息到员工档案中
```

### 3.2 月度考勤记录

```
步骤 1：search_docs(scope="knowledge", q="{{YYYY}}年{{MM}}月考勤记录")
        → 若不存在则创建，若存在则读取
步骤 2：收集当月考勤数据（出勤/请假/加班/迟到）
步骤 3：update_doc(id=考勤文档ID, content=<更新后的考勤汇总>)
步骤 4：计算异常情况并在文档中标注
步骤 5：向用户反馈考勤汇总结果
```

### 3.3 季度绩效评估

```
步骤 1：确定评估周期和参与人员
步骤 2：search_docs(scope="knowledge", q="员工档案") → 获取参评员工列表
步骤 3：create_doc(
          scope="knowledge",
          title="{{YYYY}}年Q{{N}}绩效评估",
          content=<绩效评估模板内容>
        )
步骤 4：填写各员工绩效评分、汇总排名
步骤 5：对每位参评员工，update_doc 更新其档案中的绩效记录
```

### 3.4 员工离职

```
步骤 1：search_docs(scope="knowledge", q="员工档案-{{姓名}}") → 找到档案文档
步骤 2：read_doc(id=员工档案文档ID) → 读取当前内容
步骤 3：update_doc(id=员工档案文档ID, content=<状态改为"已离职"，记录离职日期和原因>)
步骤 4：update_doc(id=人事索引文档ID, content=<更新花名册状态>)
```

### 3.5 招聘管理

```
步骤 1：创建岗位需求文档 → create_doc(title="招聘-{{岗位名称}}", content=...)
步骤 2：记录候选人信息 → update_doc 追加到"候选人追踪"文档
步骤 3：安排面试 → update_doc 追加到"面试记录"文档
步骤 4：录用 → 创建新员工档案（走入职流程）
```

### 3.6 薪资管理

```
步骤 1：search_docs(scope="knowledge", q="薪资标准") → 读取薪资标准文档
步骤 2：根据考勤和绩效数据计算实发薪资
步骤 3：create_doc(
          title="{{YYYY}}年{{MM}}月薪资发放记录",
          content=<薪资发放明细>
        )
```

## 5. 业务规则

1. **隐私保护**：员工档案含敏感信息，读取和展示时注意脱敏
2. **及时更新**：入职/离职/调岗应在3个工作日内更新档案文档
3. **考勤周期**：自然月为考勤周期，次月3号前完成上月汇总
4. **绩效公正**：绩效评估需多维数据支撑，避免主观偏见
5. **合同预警**：每月初 search_docs 检索合同到期情况，30天内到期需提醒续签
6. **数据留存**：离职员工档案文档至少保留2年，不得删除
7. **文档命名**：员工档案统一使用 `员工档案-姓名` 标题，便于 search_docs 检索
8. **索引维护**：每次创建/删除员工档案时，必须同步更新人事索引文档
