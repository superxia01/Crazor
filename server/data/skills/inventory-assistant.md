# 库存与订单管理 Skill

> 本 Skill 通过 MCP 文档工具管理全部库存与订单数据。所有 SKU、订单、库存预警、供应商信息均以文档形式存储在知识库中。

---

## 1. 可用 MCP 工具

| 工具 | 用途 | 关键参数 |
|------|------|----------|
| **create_doc** | 创建新文档 | `scope="knowledge"`, `title`, `content` |
| **update_doc** | 更新已有文档 | `id`, `title`, `content` |
| **read_doc** | 读取单个文档 | `id` |
| **list_docs** | 列出文档列表 | `scope="knowledge"`, `folder_id` |
| **search_docs** | 搜索文档 | `scope="knowledge"`, `q` |
| **read_vault_file** | 读取参考模板文件 | 文件路径 |

---

## 2. 知识库文档结构

所有库存文档存储在 `20-业务流程/60-库存管理/` 文件夹下：

| 文档类别 | 存储路径 | 文档标题格式 |
|----------|----------|-------------|
| SKU管理 | `20-业务流程/60-库存管理/` | `SKU-XXXX-产品名称` |
| SKU总览 | `20-业务流程/60-库存管理/` | `SKU总览` |
| 订单管理 | `20-业务流程/60-库存管理/` | `ORD-YYYYMMDD-XXX` |
| 订单索引 | `20-业务流程/60-库存管理/` | `订单索引` |
| 库存预警 | `20-业务流程/60-库存管理/` | `库存预警报告` / `预警历史` |
| 供应商 | `20-业务流程/60-库存管理/` | `供应商列表` / `采购记录` |
| 库存索引 | `20-业务流程/60-库存管理/` | `库存索引` |

> **搜索技巧**：使用 `search_docs(scope="knowledge", q="关键词")` 快速定位文档。例如 `search_docs(q="SKU")` 查找所有 SKU 相关文档。

## 3. 工作流程

### 3.1 SKU 注册

```
步骤 1：收集产品信息（名称、规格、成本价、售价）
步骤 2：search_docs(scope="knowledge", q="SKU总览") → 获取当前最大编号
步骤 3：分配SKU编号：SKU-XXXX（按顺序递增）
步骤 4：create_doc(
          scope="knowledge",
          title="SKU-{{XXXX}}-{{产品名称}}",
          content=<SKU卡片模板内容，填入实际数据>
        )
步骤 5：read_doc(id=SKU总览文档ID)
步骤 6：update_doc(id=SKU总览文档ID, content=<追加新SKU行>)
步骤 7：设置初始库存数量和安全库存线
```

### 3.2 订单处理

当用户说"新订单"或"收到订单"时：

```
步骤 1：记录订单信息（客户、商品、数量、金额）
步骤 2：search_docs(scope="knowledge", q="SKU-{{相关产品}}") → 检查库存
步骤 3：read_doc(id=SKU文档ID) → 确认库存是否充足
步骤 4：create_doc(
          scope="knowledge",
          title="ORD-{{YYYYMMDD}}-{{序号}}",
          content=<订单模板内容，状态=⏳待处理>
        )
步骤 5：update_doc(id=订单索引文档ID, content=<追加新订单记录>)
步骤 6：库存充足 → 建议进入发货流程
        库存不足 → 触发补货预警，更新库存预警文档
```

### 3.3 发货流程

```
步骤 1：确认发货信息（收件人、地址、物流方式）
步骤 2：search_docs(scope="knowledge", q="ORD-{{订单编号}}") → 找到订单文档
步骤 3：read_doc(id=订单文档ID) → 读取当前内容
步骤 4：update_doc(id=订单文档ID, content=<状态改为📦已发货，填入物流信息>)
步骤 5：search_docs(scope="knowledge", q="SKU-{{相关产品}}") → 找到SKU文档
步骤 6：read_doc(id=SKU文档ID) → 读取当前库存
步骤 7：update_doc(id=SKU文档ID, content=<扣减库存数量，记录出库>)
```

### 3.4 订单完成

```
步骤 1：确认客户已签收
步骤 2：search_docs(scope="knowledge", q="ORD-{{订单编号}}")
步骤 3：update_doc(id=订单文档ID, content=<状态改为✅已完成，记录签收日期>)
步骤 4：update_doc(id=订单索引文档ID, content=<更新订单状态>)
```

### 3.5 库存预警检查

当用户说"检查库存"或"库存预警"时：

```
步骤 1：search_docs(scope="knowledge", q="SKU") → 获取所有SKU文档列表
步骤 2：逐一 read_doc 读取每个SKU的库存数据
步骤 3：对比当前库存与安全库存线：
        - 库存 = 0        → 🔴紧急预警
        - 库存 ≤ 安全库存  → 🟡偏低预警
        - 库存 > 安全库存  → 🟢充足
步骤 4：search_docs(scope="knowledge", q="库存预警报告") → 找到预警文档
步骤 5：update_doc(id=预警文档ID, content=<最新的预警数据>)
        若不存在则 create_doc(title="库存预警报告", content=...)
步骤 6：向用户反馈预警汇总和建议补货数量
```

### 3.6 入库操作

```
步骤 1：search_docs(scope="knowledge", q="SKU-{{产品名称}}")
步骤 2：read_doc(id=SKU文档ID) → 读取当前库存
步骤 3：update_doc(id=SKU文档ID, content=<增加库存数量，记录入库>)
```

### 3.7 供应商管理

```
步骤 1：search_docs(scope="knowledge", q="供应商列表")
步骤 2：创建/更新供应商信息文档
步骤 3：采购下单后，更新采购记录文档
```

## 5. 订单状态流转

```
创建订单(⏳待处理) → 已发货(📦) → 已完成(✅)
       ↓                ↓
    已取消(❌)        退换货(⚠️)
```

**状态更新规则**：
- 创建订单时状态为 `⏳待处理`
- 填入物流单号后状态改为 `📦已发货`
- 客户签收确认后状态改为 `✅已完成`
- 每次状态变更必须更新操作日志
- 取消订单时，若已扣减库存需同步恢复

---

## 6. 业务规则

1. **SKU编号**：按创建顺序递增，格式 `SKU-XXXX`
2. **库存实时性**：每次出/入库必须立即 update_doc 更新库存数
3. **安全库存**：根据历史销量设定，建议为周均销量的2-3倍
4. **订单编号**：格式 `ORD-YYYYMMDD-XXX`，按日期+当日序号
5. **预警频率**：每次用户查询库存时自动检查预警
6. **供应商管理**：记录主要供应商联系方式和合作条款
7. **文档命名**：SKU文档统一使用 `SKU-XXXX-产品名称` 标题，便于 search_docs 检索
8. **索引维护**：每次创建/更新订单或SKU时，必须同步更新对应的索引文档
