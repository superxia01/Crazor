# Crazor — 产品需求文档 (PRD)

> 开源版 WorkBuddy：以 Hermes Agent 为引擎，Web 浏览器为主要入口，打通 20+ IM，技能一键安装，MCP 连接万物，3D 虚拟办公室沉浸办公。

---

## 1. 产品概述

### 1.1 产品定位

Crazor 是一款**开源的 AI Agent 桌面/Web 工作台**，面向中小企业员工和非技术用户，提供"开箱即用"的 AI 助手体验。

**一句话描述：** 把 Hermes Agent 的强大能力（自我学习、20+ IM、70+ 工具、MCP 生态）包装成普通人能用的 GUI 产品。

### 1.2 核心差异化

| 维度 | Crazor | 腾讯 WorkBuddy | OpenClaw |
|------|--------|---------------|----------|
| **开源** | MIT 开源，可商用 | 闭源 | GPL-3.0，商用受限 |
| **定价** | 自托管，仅付 API 费用 | 198-316 元/人/月 | 免费，API 费用自理 |
| **模型** | 18+ provider，任意切换 | 混元/DeepSeek/GLM | 任意模型 |
| **IM** | 20+ 平台，无生态限制 | 深度绑定腾讯生态 | 有限 IM 支持 |
| **生态** | MCP + Skills Hub 8+ 来源 | SkillHub + OpenClaw 兼容 | ClawHub |
| **部署** | 本地/云/Serverless 任选 | SaaS 或专有云 | 本地部署 |
| **数据** | 完全自主可控 | 经腾讯网关 | 本地存储 |
| **GUI** | Web + Tauri 桌面 | 桌面客户端 | CLI 为主 |

### 1.3 目标用户

**主要用户：** 中小企业员工、自由职业者、跨境电商卖家、内容创作者

**用户画像：**
- 不懂技术，但需要 AI 帮忙处理日常办公任务
- 通过飞书/微信等 IM 跟 AI 助手沟通
- 需要连接 Gmail、Shopify、小红书、公众号等外部平台
- 希望一个工具搞定所有事，不想在多个 App 之间切换

### 1.4 使用场景

1. **移动办公**：在飞书里发消息"帮我查一下今天的 Shopify 订单"，Hermes 查询后回复
2. **文档处理**：浏览器打开 Crazor，说"把这份 Excel 生成周报 PPT"，自动完成
3. **社媒运营**：在 Web 界面配置小红书连接器，让 AI 自动生成并发布内容
4. **团队协作**：团队成员通过各自 IM 跟同一个 Hermes 实例交互，共享技能和连接器
5. **3D 沉浸办公**：在虚拟办公室里，点击不同工位进入不同工作台

### 2.2 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| **前端框架** | React 19 + Vite | 与 Slate-Desk/hermes-desktop 一致，可直接复用组件 |
| **UI 组件** | shadcn/ui + Radix UI | Slate-Desk 已用，可定制性强 |
| **样式** | Tailwind CSS 4 | 两个参考项目都用，效率高 |
| **Markdown 编辑器** | Milkdown | Slate-Desk 已集成，支持数学/代码/Mermaid |
| **3D 渲染** | Three.js / React Three Fiber | hermes-desktop Claw3D 使用 |
| **终端模拟** | xterm.js | 两个参考项目都已验证 |
| **后端 API** | Python FastAPI | 与 Hermes 同语言，可直接调用内部模块 |
| **实时通信** | SSE streaming | hermes-desktop 已验证此方案 |
| **数据库** | PostgreSQL (生产) / SQLite (开发) | 多用户并发需求 |
| **缓存/队列** | Redis | 会话状态、消息队列 |
| **文件存储** | MinIO (S3 兼容) | 对话附件、文档、导出文件 |
| **容器化** | Docker Compose → K8s | Hermes 已有 Docker 后端支持 |

### 2.3 二开基础

| 项目 | 用途 | 处理方式 |
|------|------|----------|
| **Hermes-Slate-Desk** | 主要 UI 基础 | Fork，提取 React 前端，去掉 Tauri 依赖，改为 Web SPA |
| **hermes-desktop** | Claw3D 虚拟办公室 | Fork，抽取 Office/Claw3D 模块，适配 Web |
| **hermes-agent** | 核心引擎 | 直接用 pip 安装，不改源码（除非需要定制） |

### 2.4 Tauri 桌面版（Phase 5）

Phase 1-3 完成后，同一套 React 代码包 Tauri shell：

```
同一套前端代码
├── Web 模式: Vite dev server → 浏览器访问
└── Desktop 模式: Tauri 2 shell → 原生窗口
    └── 额外能力: 本地文件系统、系统通知、托盘图标、开机自启、离线模式
```

### 9.2 AI 笔记与 Obsidian 兼容（2026-05-19）

**背景：** 当前 AI 笔记使用 Milkdown（React Markdown 编辑器），数据存 localStorage。能力上与 Obsidian 差距较大：缺少双向链接、关系图谱、标签系统、插件生态等。

**方向：**
1. **存储层兼容**：笔记从 localStorage 迁移为本地 Markdown 文件，使用 frontmatter 存储元数据（标题、标签、创建时间），确保 Obsidian 可以直接打开工作区目录
2. **双向链接**：后续在编辑器中支持 `[[]]` 语法和反向链接面板
3. **知识图谱**：基于链接关系生成简单的可视化图谱
4. **格式互通**：Obsidian 用户可以选择用 Obsidian 编辑笔记，Crazor 中也能看到变更

**当前笔记技术栈：**
- 编辑器：`@milkdown/crepe` + `@milkdown/kit`（WYSIWYG Markdown）
- 存储：localStorage（JSON 序列化）
- UI 框架：React 组件
